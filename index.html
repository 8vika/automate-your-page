Lesson 1: The Basics of the Web and HTML

How the Web Works

Essentially, the World Wide Web is a network that connects computers with servers. The computers "talk" to servers by sending something called HTTP (Hypertext Transfer Protocol) requests.

HTML

HTML or Hypertext Markup Language is the standard markup language used to create web pages. It is written in the form of HTML elements consisting of tags enclosed in angle brackets.

Tags and Elements

An HTML tag is always contained within angled brackets. Most tags have an opening tag and a closing tag. An element refers to everything within a set of opening and closing tags.

Computers are smart: people are stupid!

Unlike humans, computers follow instructions exactly as they are given. So long as the syntax is correct computers will do precisely what they are told. Humans, on the other hand, not so much.

Inline vs Block Elements

Inline elements format everything in a single line. Examples are: br (break), em (emphasis), and img (image). Box elements format the contents within an invisible textbox. Examples are: p (paragraph) and div (divide).

Lesson 2: Creating a Structured Document with HTML

Developer Tools (in the Browser)

The browser provides many useful tools for analyzing or creating a webpage. For example, many of the attributes such as borders, color, font size, etc., can be changed interactively and the result can be viewed in real time.

The "tree-like structure" of HTML

HTML and CSS (Cascading Style Sheets) are both languages where HTML controls the placement of content and CSS controls the style (decoration) of content. Java script is used to control the interactive components on the page. These languages are organized using something called DOM or Document Object Model into a hierarchical structure resembling a tree.

The relationship between indented HTML and boxes

The tree structure found in DOM hierarchy is viewed in text as nested indentations and viewed on the webpage as nested boxes.

Text Editors (for programming)

Although html code can be written in a word processor it is not recommended because there are many useful programming text editors that provide feedback in real time like Sublime, for example.

Lesson 3: Adding CSS Style to HTML Structure

Avoiding Repetition

Avoiding repetition is important for a variety of reasons.

Avoiding Errors: CSS allows a particular style to be applied to several different elements by assigning a class. This saves time, space, and helps reduce coding errors by eliminating the need for repitition. Another property called inheritance governs rules which eliminate the need for certain styles to be declared repeatedly.

Consistency: In this web page, I wanted every lesson to look the same. Instead of manually giving each one the same style, I was able to assign each lesson div the same class and then specify the style for that class in only one place! By avoiding repetion, it's now very easy for me to make changes to this style.

CSS

CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.

Lesson 4: A Deeper Understanding

Understanding Programming

To become proficient at programming one must first learn to think like a programmer. Thinking like a programmer requires more than just investing in a pair or thick glasses and a pocket protector. To think like a programmer one must master three things: one, procedural thinking; two, technological empathy; and three, abstract thinking.

Computer languages are similar to spoken language in that they employ a set of grammatical rules referred to as syntax. Python, for example, uses the Backus Normal Form notation technique where commands are given in the form of expression, operator, expression.

Some common "got-cha's" and other points to be mindful of in Python are: 1)Always introduce a variable before you use it. 2)Avoid programming bugs by following the convention of never capatalizing variables. 3)String variables must have the same number of quotes on both sides. 4)In addition to using single or double quotes you can also use tripple quotes to create a multi-line string. 5)The plus operator is used to concatenate two strings but can't combine a string and a number. 6)The multiply operator can be used to repeat a string. 7)A single equal operator means assignment and a double equal operator means equality. 8)When Python code crashes the message it gives you is called a "traceback." The traceback tells you what line crashed and the file that was running. 9)To obtain a precise answer when dividing use a decimal place to invoke floatinig point division.

Computational Problem Solving

Abstract thinking is the lion's share of solving any computational problem because you must first understand the problem. Such problems are defined by possible inputs, possible outputs, and the relationship between them. So a "solution" is a procedure that takes the inputs and produces the desired outputs. Many beginners try to immediately translate the problem into code in their heads and get frustrated. Verterans, on the other hand, employ tools such as psuedo code, simple mechanical algorithms, and think in terms of valid inputs and outputs before attempting to write any code.

Pythonista's Guide to All Problems in the Galaxy can be summarized as follows:

0. Don't Panic!

1. What are the inputs?

2. What are the outputs?

3. Work through some examples by hand.

4. Come up with a simple mechanical solution.

5. Develop and test as you go.

Understanding Functions

Functions are critical to eliminating repetition. Python has many built in functions, but the true value is being able to define your own functions to perform a particular repetitive task. When making functions watch for syntax errors, spelling errors, indentation errors, and be aware that calling a procedure without a return does nothing.

An Introduction to Lists: Mutation versus Aliasing and List Operators

There are two major differences between lists and strings: 1) Strings contain only characters whereas lists can contain anything. 2) Lists can support something called Mutation.

Mutation sounds like the title of a low budget science fiction B-movie. However, in computer science, mutation means you can change the value of a list after you have created it. For example, the append function mutates the old list by adding an element to the end. Unlike in B-movies, this change requires no radiation or nuclear explosions. Conversely, using the plus operator introduces a new list.

Aliasing means there are two different ways to refer to the same object.

The len function, returns the number of elements in the list. The index operator, returns the first position where the value is found in the list otherwise it produces an error. The in operator returns true if the value is contained in the list.
